UI/UX Case study -IKEA Application

Overview

We know that IKEA is the largest furniture company in the world. This company has lots of products making people easy to use and build on their own in some product but why their app is hard to use and difficult to understand that's why I redesign the IKEA app to make the app easy and useful for people

App version

3.9.1

Time taken:1 months

The Challenges

People who use the IKEA app have a better chance to figure out what's in my design therefore they can use it easier before it was! this redesign include creating a new flow and design for users to search for items by word by word or by filtering and adding the selecting item in the cart saved in your heart list so users get easier to use and make them happier

Design Sprint

The design is carried out in this research using design thinking methodology

Step-1 : Empathize

Observation

When I observed IKEA version 3.9.1 app where i feel less intraction and users can't be understand how to use. So i noted a few points that could solve the problem in this app so that people can interact with this Application more peacefully and not to frustrated while using this app

Research Plan

Background

The users review in Google Play store totally went negative people who uses IKEA app tends to handle difficult to use the app and hard to search any item in it.Therfore we the UX designer needs to evaluate the current user interface to find out the problem that amke it difficult for users to serach items,lets find the solution to the problems by creating a new user interface that makes it easier for users.

Step2-Define

User Persona

I created user personas from 5 interviewed users after running Usability testing using useberry User personas are needed to descirbe potential users based on the research that has been done. The user persona that I‚Äôve created describes user profile and personal Background, User Environment, Pyschographics, end goals and scenario of using the application, user behavior using the application, frustration when using the application.

Identifying the Problem

Based on the results of the analysis on usability testing. The app face 4 main problems by users

Detail: Home Screen
Problem:

Users can't understand what is in the home screen the picture in the home screen is so big and it is difficult to navigated where to go? The steps that the users must take are not simple because the home screen should be as accessible as possible for the users to understand.

Solution:

Instead of covering half of the screen we can use slider in the home screen so that users can understand even better and navigate easily and can see rest of the products. its user interface is not good

Problem-2

Detail: Home screen scrolling part
Problem:

The many problems rise in the home screen section in that home screen there is no product detail, rating, price, etc all was the only picture and lots of pictures like gallery

Solution:

The solution for my design is to separate all the products with a small card and with sliders add some product images and detail to it.

Problem-3

Detail: Cart

Item cart size item price, item cart

Problem:

The card size and the item price are too large in the item details the add to cart button is not properly fixed it looks unbalanced because on-screen looks full and the information must be beautiful neatly aligned

Solution:

Change the item image size and item price font to a smaller size than before to create a more balanced design. The solution can also make more information and people can buy easier than before the position of the cart button has been changed and added trash button and quantity for the product in the cart section at the bottom section you can see how much product added in the cart section and the amount you need to pay then checkout

Problem-4

Detail: Wishlist /Add to favourite
Problem:

Users have difficulty finding shopping list menu and detail in the wishlist just a image on the wishlist item it look unbalanced design in the my favorites

Solution:

Changed the images added price, review starts product name and a heart icon so that users can more easlity fint the list on my favorites

UI Design

Homepage

The homepage is that will first appear when the user opens the application The changes made in the homepage menu are placing the search button on the center.Then the next changes is the banner converted into slider and preference at the top so that users can filter there need what they want in the home screen next i added notification icon and mic icon to the top-right of homepage.

My Profile

In the Profile section, I have changed everything added the user's picture, address, email id is given. Then the color full section is added so that users can identify which to click and which to log out.

My Cart

The user can see the items in the cart the cart have a trash icon to delete instanstensly and increase or decrease the product quantity for the user the total purchase for amount and a checkout direct to the payment area. all were added in my design

Step-5 Test

Usability Testing After Redesign

After redesigning the application, I re-evaluated the new design that I had created I tested 5 users who participated in my task scenario. this test is carried out using Figma prototype and useberry and users are aked to run the prototype according to the task on their device. This test also wanted to see how the user‚Äôs expectations of the new design that I created, and find out whether the solution I implemented had addressed all the pain points that users felt in the first test of the current IKEA application.

Test Results

The results of the usability test of the new design got more positive results and feeback All users can perform every task and the users using the app got excited and happy to use and they told me a quite error to change and I've have changed after the usability test. However, there are still some improvements that I have to make better than new design there are new opportunities that can be developed futher.

Conclusion

Overall, users have a nice experience after trying out the new design prototype.
